# Agent gim 56

Одностраничное веб-приложение для тренировки химических названий в формате карточек.

- **Лицевая сторона:** формальное название / формула.
- **Обратная сторона:** тривиальное (бытовое) название.
- **Опционально:** изображение вещества (`AI`, `Wiki`, `2D` или `3D` структура).

Приложение полностью находится в файле `index.html` и работает в браузере без сборки и серверной части.

## Возможности

- Готовые кнопки для заполнения списка:
  - `Готовый: органика`;
  - `Готовый: неорганика`.
- Кнопка `Случайные 10 из базы` для теста на основе встроенного справочника `BASE_CARDS`.
- Пользовательский список веществ:
  - ввод по строкам или через запятую;
  - формат пары `формальное | тривиальное`;
  - можно указывать только одну часть.
- Автодополнение карточек перед стартом:
  - при отсутствии `formalRu` — попытка восстановить через Mistral;
  - при отсутствии `trivialRu` — попытка восстановить через Mistral;
  - попытка найти PubChem CID через перевод RU→EN и PubChem PUG REST.
- Источники изображений:
  - `ai` — генерация через Pixazo;
  - `wiki` — миниатюра из Wikipedia API;
  - `structure2d` — 2D структура из PubChem PNG;
  - `structure3d` — 3D структура из PubChem PNG.
- Кэширование изображений в памяти на время сессии.
- Сохранение/удаление пользовательских пресетов в `localStorage`.
- Локальное хранение API-ключей в `localStorage`.

## Как запустить

### Вариант 1: локально

1. Откройте `index.html` в браузере.
2. При необходимости введите API-ключи Mistral/Pixazo.
3. Выберите источник карточек и нажмите **«Начать тест (10 карточек)»**.

### Вариант 2: статический хостинг

Опубликуйте `index.html` как статический файл (например, GitHub Pages / Netlify / Vercel static).

## Интерфейс и сценарий работы

1. **Ключи API**
   - `Mistral (названия)` — для автодополнения названий и генерации описания внешнего вида вещества для AI-картинок.
   - `Pixazo (картинки AI)` — для режима `AI-рендер`.

2. **Набор карточек**
   - Можно выбрать случайные карточки из базы.
   - Можно подставить готовый список `органика` или `неорганика`.
   - Можно вручную ввести собственный список и сохранить его как пресет.

3. **Изображения**
   - Переключатель `Включить изображения`.
   - Тип изображения: `AI-рендер`, `Wiki-фото`, `2D-структура`, `3D-структура`.

4. **Старт теста**
   - Приложение формирует пул ровно из 10 карточек:
     - если в списке больше 10 — случайно выбирает 10;
     - если меньше 10 — добирает из встроенной базы.
   - Затем обогащает карточки (названия, CID) и перемешивает порядок.

5. **Прохождение**
   - Кнопки: `Назад`, `Перевернуть`, `Вперед`.
   - На обратной стороне показывается только ответ (без картинки).

## Формат пользовательского списка

Поле «Свой список» принимает строки через перевод строки или запятую.

Примеры:

- Полная пара:
  - `фениламин | анилин`
- Только формальное:
  - `этановая кислота`
- Только тривиальное:
  - `| уксусная кислота`

Как формируется итоговый набор:

- >10 строк: случайные 10;
- <10 строк: недостающие карточки добавляются из `BASE_CARDS`.

## Работа с API-ключами

### Какие ключи нужны

- **Mistral** — автогенерация недостающих названий + текстовое описание для AI-промпта.
- **Pixazo** — генерация изображений в режиме `AI-рендер`.

Если ключей нет, приложение все равно работает, но:

- Mistral-запросы не выполняются;
- AI-картинки недоступны;
- `Wiki`/`2D`/`3D` режимы могут работать без ключей.

### Где хранятся ключи

В `localStorage` браузера:

- `chem_quiz_api_secrets_v1`

## Деплой на GitHub Pages (подстановка ключей)

В `index.html` есть плейсхолдеры:

- `__CHEM_BUILD_MISTRAL__`
- `__CHEM_BUILD_PIXAZO__`

Их можно заменить в CI/CD, чтобы не вводить ключи вручную в интерфейсе.

Пример:

```bash
sed -i "s|__CHEM_BUILD_MISTRAL__|$MISTRAL_API_KEY|g" index.html
sed -i "s|__CHEM_BUILD_PIXAZO__|$PIXAZO_API_KEY|g" index.html
```

## Архитектура

Приложение написано на чистом HTML/CSS/JavaScript и включает:

- UI-блоки:
  - настройки ключей;
  - выбор/редактирование списка веществ;
  - управление изображениями;
  - экран квиза.
- Встроенную базу `BASE_CARDS` с полями `formalRu` и `trivialRu`.
- Логику:
  - парсинг строк формата `формальное | тривиальное`;
  - сбор пула из 10 карточек;
  - обогащение через Mistral/перевод/PubChem;
  - загрузку изображений;
  - рендер карточки и навигацию.

### Используемые внешние API

- Mistral Chat Completions (`mistral-small-latest`);
- Pixazo SDXL API;
- Google Translate endpoint;
- PubChem PUG REST;
- Wikipedia API (`pageimages`).

---

## Данные и приватность

- Пресеты и API-ключи сохраняются локально в браузере (`localStorage`).
- Серверной БД в проекте нет.
- При включении сетевых режимов названия веществ отправляются во внешние API.

---

## Ограничения

- Доступность внешних API зависит от CORS, лимитов и состояния сервисов.
- `structure2d`/`structure3d` доступны не для всех веществ.
- Качество Mistral/Pixazo зависит от модели, тарифа и доступности API.
- В приложении нет режима подсчета баллов: это карточечный тренажер.

---

## Траблшутинг

- **Не грузятся AI-картинки**
  - Проверьте Pixazo-ключ и выбранный тип `AI-рендер`.

- **Не подставляется второе название в паре**
  - Проверьте Mistral-ключ или укажите обе части вручную (`формальное | тривиальное`).

- **Нет 2D/3D-структуры**
  - Попробуйте `Wiki`/`AI-рендер` или более точное название.

- **Пропали сохраненные данные после перезапуска браузера**
  - Убедитесь, что браузер не очищает `localStorage` автоматически и сайт открыт на том же origin.
