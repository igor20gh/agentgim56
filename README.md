# Agent gim 56

Одностраничное веб-приложение для тренировки химических названий в формате карточек:

- лицевая сторона: формальное название вещества;
- обратная сторона: тривиальное/бытовое название;
- опционально: изображение вещества (AI-рендер, фото из Wikipedia или 2D-структура из PubChem).

Приложение полностью находится в файле `index.html` и работает в браузере без сборки и серверной части.

## Возможности

- Готовые наборы карточек:
  - органика (RU);
  - неорганика (RU);
  - ИЮПАК органика (EN);
  - ИЮПАК неорганика (EN).
- Случайные 10 карточек из встроенной базы.
- Пользовательский список веществ:
  - построчно или через запятую;
  - поддержка пар `формальное | тривиальное`;
  - отдельный режим для англоязычного ИЮПАК.
- Автодополнение неполных данных:
  - перевод через Google Translate endpoint;
  - получение CID из PubChem;
  - генерация недостающего формального/тривиального RU-названия через Mistral.
- Изображения по выбору:
  - `ai` (Pixazo SDXL);
  - `wiki` (Wikipedia API pageimages);
  - `structure2d` (PubChem PNG по CID/названию).
- Кэширование изображений в памяти на время сессии.
- Сохранение и удаление пользовательских пресетов (`localStorage`).
- Локальное сохранение API-ключей (`localStorage`).

## Как запустить

### Вариант 1: локально, без сервера

1. Откройте `index.html` в браузере.
2. Заполните API-ключи при необходимости.
3. Выберите набор карточек и нажмите **«Начать тест (10 карточек)»**.

### Вариант 2: любой статический хостинг

Достаточно опубликовать `index.html` как статический файл (GitHub Pages, Netlify, Vercel static и т.д.).

## Интерфейс и сценарий работы

1. **Ключи API**  
   Введите ключи Mistral и/или Pixazo и сохраните.

2. **Набор карточек**  
   Выберите готовый пресет или введите свой список веществ.  
   Можно сохранить текущий список как пользовательский пресет.

3. **Изображения**  
   Включите/выключите картинки и выберите тип источника:
   - `AI-рендер`;
   - `Wiki-фото`;
   - `2D-структура`.

4. **Старт теста**  
   При запуске приложение:
   - строит пул ровно из 10 карточек (дополняет из базы при нехватке);
   - обогащает карточки названиями/CID/переводами;
   - перемешивает порядок.

5. **Навигация**  
   Используйте кнопки:
   - **Назад**;
   - **Перевернуть**;
   - **Вперед**.

## Формат ввода списка веществ

Поле «Свой список» принимает строки через перевод строки или запятую.

### RU-режим (галочка ИЮПАК выключена)

- Полная пара:
  - `фениламин | анилин`
- Только формальное:
  - `этановая кислота`
- Только тривиальное:
  - `| уксусная кислота`

### EN ИЮПАК-режим (галочка ИЮПАК включена)

- Полная пара:
  - `ethanoic acid | acetic acid`
- Только ИЮПАК:
  - `phenylamine`
- Только бытовое EN:
  - `| baking soda`

### Как формируется итоговый набор

- Если строк больше 10: случайно выбираются 10.
- Если строк меньше 10: недостающие добираются из встроенной базы.
- Для неизвестных/неполных пар приложение пытается:
  - найти совпадение во встроенной базе алиасов;
  - получить недостающие данные через перевод/Mistral/PubChem.

## Работа с API-ключами

### Какие ключи нужны

- **Mistral** — для генерации недостающих названий и текстового описания внешнего вида вещества.
- **Pixazo** — для режима изображений `AI-рендер`.

Если ключей нет:

- приложение все равно работает;
- часть автодополнения и AI-рендер становятся недоступны;
- `Wiki` и `2D-структуры` могут продолжать работать.

### Где хранятся ключи

Ключи сохраняются только в `localStorage` текущего браузера:

- `chem_quiz_api_secrets_v1`

## Деплой на GitHub Pages (с подстановкой ключей)

В `index.html` предусмотрены плейсхолдеры:

- `__CHEM_BUILD_MISTRAL__`
- `__CHEM_BUILD_PIXAZO__`

При деплое можно заменить их на реальные значения (например, шагом CI), чтобы не вводить ключи вручную в UI.

Пример подстановки на Linux runner:

```bash
sed -i "s|__CHEM_BUILD_MISTRAL__|$MISTRAL_API_KEY|g" index.html
sed -i "s|__CHEM_BUILD_PIXAZO__|$PIXAZO_API_KEY|g" index.html
```

Важно: если храните ключи в CI/CD, используйте секреты репозитория и не коммитьте реальные токены в git.

## Архитектура

Приложение написано на «чистом» HTML/CSS/JavaScript и состоит из:

- UI-блоков:
  - настройки ключей;
  - выбор и редактирование пресетов;
  - панель квиза;
  - блок изображений.
- Встроенной базы `BASE_CARDS`:
  - RU формальное;
  - RU тривиальное;
  - PubChem CID;
  - IUPAC EN + EN алиасы.
- Логики:
  - парсинг пользовательского списка;
  - нормализация и поиск по алиасам;
  - обогащение карточек (перевод, CID, Mistral);
  - загрузка изображений по выбранному источнику;
  - рендер и навигация по карточкам.

### Основные внешние API

- Mistral Chat Completions API (`mistral-small-latest`);
- Pixazo SDXL image API;
- Google Translate endpoint;
- PubChem PUG REST;
- Wikipedia API (`pageimages`).

---

## Данные и приватность

- Пресеты и ключи сохраняются локально в браузере (`localStorage`).
- Серверной БД в проекте нет.
- При включенных сетевых функциях названия веществ отправляются во внешние API согласно выбранному режиму.

---

## Ограничения

- CORS/доступность внешних API может отличаться в разных средах.
- Режим `structure2d` работает не для всех веществ (нужен CID или корректное имя).
- Качество результатов Mistral/Pixazo зависит от модели, тарифов и доступности сервисов.
- Приложение не ведет учет «правильно/неправильно» и не считает баллы — это тренажер карточек, а не экзаменационный тест.

---

## Траблшутинг

- **Не грузятся AI-картинки**  
  Проверьте ключ Pixazo и выбранный тип `AI-рендер`.

- **Нет тривиального/формального названия**  
  Проверьте ключ Mistral или добавьте полную пару в пресете вручную.

- **Не появилась 2D-структура**  
  Попробуйте `Wiki`/`AI-рендер` или укажите более точное название (желательно IUPAC EN).

- **После перезапуска браузера пропали данные**  
  Убедитесь, что браузер не очищает `localStorage` автоматически и сайт открыт в том же origin.
